## Update Cadence

### How Often Does the Source Data Change?
- **USDOT publishes new NAD releases quarterly** (Q1, Q2, Q3, Q4)
- Each release is a full replacement file (~7.9 GB ZIP, ~38 GB extracted)
- Official release page: https://www.transportation.gov/gis/national-address-database/nad-disclaimer-page-0

### Is the Update a Delta (Incremental)?
**No — USDOT publishes full replacement files, not deltas.**
Each quarterly ZIP contains ALL 96M+ addresses, not just changes.

### Recommended Update Strategy

```bash
# Check for new release quarterly (run this as a cron or reminder)
node nad/download.js --dry-run

# Full re-import when new release is available (~4-6 hours)
node nad/download.js --resume   # downloads new ZIP, re-imports
```

### What Changes Between Releases?
- New construction gets official addresses assigned
- Rural Route / Range Road addresses get rooftop coordinates added
- Incorporated municipality boundaries change (city annexations)
- Neighborhood Nbrhd_Comm names updated by local governments
- Address decommissions (demolished buildings, rural consolidations)
- ~2–5% of records change per quarter based on historical NAD deltas

### Practical Recommendation
| Update Frequency | If You Are... |
|-----------------|---------------|
| **Quarterly** (every 3 months) | General address verification, mortgage, PropTech |
| **Semi-annual** (every 6 months) | Lower-stakes use cases, analytics, market research |
| **Annual** | Genealogy, historical analysis, slowly-changing use cases |
| **Never skip** | 911/emergency response, insurance underwriting, fraud detection |

### Delta Workaround (If Needed)
USDOT does not publish deltas, but you can **build your own**:
```sql
-- After re-import, compare nad_uuid across versions
-- nad_uuid is a stable identifier per address record
SELECT nad_uuid FROM addresses WHERE date_update > '2026-01-01'
```
The `date_update` column on each address record tells you when
the local authority last modified that specific address. Filter
on this to find recently-changed records within a full import.
