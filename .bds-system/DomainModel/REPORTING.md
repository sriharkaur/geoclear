# BDS Entity Lineage Reporting
> Report templates, visibility formats, and COMMS integration for lineage and delivery status.
> Last updated: 2026-04-15 | Version: 1.0

---

## Report Types

| Report | Trigger | Audience | Frequency |
|--------|---------|---------|-----------|
| Lineage Trace | On demand or when entity state changes | Engineer/PM | Ad hoc |
| Goal Status Dashboard | `/business-goal status` or `/cpm` | Founder/CPM | Weekly |
| Delivery Chain Report | Every 5 tasks completed or after deploy | CPM/Engineering | Per milestone |
| KPI Pulse | KPI measurement cycle | Founder/CMO/CPM | Weekly |
| Orphan Audit | `/domain-lineage audit` | CPM/Architect | Every 2 weeks |
| COMMS Lineage Block | Every COMM item creation | User | Per COMM item |

## COMMS Lineage Block Format

Every COMM item that relates to an entity in the delivery chain MUST include this block:

```markdown
**[LINEAGE]** BI-2026-0001 › VIS-2026-0001 › BG-2026-0001 › DG-2026-0001 › EPIC-2026-0001 › FEAT-2026-0001 › REQ-2026-0008 › DESIGN-2026-0012 › TASK-2026-0047 ⏳

**Impact on goal:** BG-2026-0001 ("$10K MRR by June 30") is affected.
  TASK-2026-0047 is blocking FEAT-2026-0001 completion.
  FEAT-2026-0001 is 1 of 5 required for DG-2026-0001.
  DG-2026-0001 progress: 2/5 features → contributes ~40% of BG-2026-0001 engineering value.
```

Rules:
- Use `›` as chain separator (not `/` or `→`)
- Append status symbol to the current (deepest) entity
- Include the "Impact on goal" block only when the COMM item is a BLOCKER or REVIEW category
- For FYI items: lineage tag only, no impact block required

## Integration with Existing BDS Skills

| Skill | Lineage actions |
|-------|----------------|
| `/bds` Layer 1 | Runs orphan count; flags if >5% orphan rate |
| `/bds` Layer 4 | Shows BG progress with KPI pulse for each |
| `/business-goal status` | Produces R2 + triggers R4 |
| `/cpm` | Full R2 + R3 for current quarter |
| `/dev-requirements` | Adds `feature`, `epic`, `dev_goal`, `business_goal` to REQ frontmatter |
| `/dev-plan` | Adds `business_goal` and `feature` to every TASK |
| `/dev-deploy` | Creates DEP entity, links included TASKs |
| `/dev-verify` | Creates VR, links to DEP; if fail → creates INC |
| `/domain-lineage` | Dedicated lineage tool (see skill) |
